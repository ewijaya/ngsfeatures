8 * Purpose: Compute the connected components of a graph.
9 *
10 * Usage: Construct with size of graph. Then add edges one at a time with addEdge() or
11 * add a whole graph with addGraph(). addEdge() has the advantage that the connected
12 * components can be computed even if all the edges don't fit in memory at the same time.
13 *
14 * Output: getNodeComponents( FLEArray<nodeIndexT>& nodeComponents );
15 * assigns nodeComponents[i] with a component number such that all nodes in the
16 * same component as i have the same component number.
17 *
18 * The order of the componenent numbers is arbitrary but assigned
19 * so that the numbers 0,1,2,.. are used up to one minus the total number of
20 * connected components.
21 *
22 * Requirements: space: linear in the number of vertices v. (not edges!)
23 * time: O( vLogv ) I think. Could be made almost linear if disjoint set was
24 * more carefully implemented. In particular "path compression" is used but
25 * "union by rank" is not.
26 *
27 * Reference: Uses Disjoint-set forests algorithm.
28 * For example see "Introduction to Algorithms" by C, L, & R.
29 */
