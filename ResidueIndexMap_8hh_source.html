216=head1 SYNOPSIS
217
218Class for mapping characters (representing residues in biosequences) to serial numbers.
219
220=head1 DESCRIPTION
221
222B<ResidueIndexMap> is a class for the common task of mapping characters to
223indices for array access. For example mapping the DNA bases (a,c,g,t) to
224(0,1,2,3) etc. The characters are referred to as "residues" and the integers
225they map to as "residue indices".
226
227This class only supports a one to one mapping between residues and
228integers starting with 0.
229
230A subset of the residues can be designated as representing "proper" residues.
231A typical use of this is to distinguish between characters which represent
232actually residues and others. Using DNA as an example, {a,c,g,t} would be
233proper residues but "n" would not.
234
235
236=head1 types
237
238 typedef char residueT;
239 typedef unsigned char indexT;
240 typedef FLENumArray<indexT> arrayT;
241 typedef std::vector<indexT> vectorT;
242
243A residue index is of type B<indexT>, a sequence of residue indices can be
244contained in a B<arrayT> or a B<vectorT>. The type of a residue, B<residueT>
245is also implicit in the method signatures which use B<std::string> to represent
246a sequence of residues.
247

249=head1 Constructors
250
251 ResidueIndexMap( const std::string& alphabet,
252 const bool& caseSensitive = false ) throw( std::invalid_argument );
253
254 ResidueIndexMap( const std::string& alphabetProper,
255 const std::string& alphabet,
256 const bool& caseSensitive = false ) throw( std::invalid_argument );
257
258 ResidueIndexMap( const char* const& alphabetProper,
259 const char* const& alphabet,
260 const bool& caseSensitive = false ) throw( std::invalid_argument );
261
262The arguments given respect follow two conditions:
2631. I<alphabetProper> must be a prefix of I<alphabet>, (it is set to I<alphabet> if omitted).
264
2652. The characters in I<alphabet> must each be unique. The position of each residue
266in I<alphabet> is the residue index assigned the residue.
267
268A B<std::invalid_argument> exception is thrown if the arguments violate those conditions.
269
270
271If constructed with I<caseSensitive> = true, characters, e.g. ("a", "A"), which
272differ only in case will be considered equivalent.
273
274
275The constructors are easily illustrated with an example. The constant
276DNAExtendedResidueIndexMap, defined in
277L<ResidueIndexMapPredefinedAlphabets>, is constructed by
278
279 ResidueIndexMap( "acgt", "acgtbdhkmnrsvwy" )
280
281Note that first argument is a prefix of the second argument. This
282is required. If the second argument is omitted all residues are
283considered to be proper.
284
285

287constructor from a binary stream are provided.

465methods above. This would probably be desirable for alphabet sizes >> 10, especially
466if those methods are found in inner loops.
