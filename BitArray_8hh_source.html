2 * Author: Paul B. Horton
3 * Organization: Computational Biology Research Center, AIST, Japan
4 * Copyright (C) 2008, Paul B. Horton, All rights reserved.
5 * Creation Date: 2003.6.5
6 *
7 * Purpose: Provide an efficient bit array.
8 *
9 * Usage: The operator [] is overridden to access elements. BitArray[i] returns the bit at position i.
10 * However BitArray[] is NOT a valid lvalue. To asign values use BitArray.set( size_t i, bool val ) or
11 * just BitArray.set( i ) to set to 1 and BitArray.clear( i ) to set to 0.
12 * Normally the index i is checked for overflow. However there are two ways
13 * to avoid this small overhead -- one is to use the methods clearFast, and setFast.
14 *
15 * The more complete way to avoid this overhead however is to use the derived class BitArrayFast. This
16 * class does not check indices for overflow unless (for debugging) the macro
17 * BITMATRIX_ALWAYS_CHECK_INDICES is defined.
18 *
19 * You may iterator over true values using a BitArray::trueIterator iterator
20 * with trueBegin() and trueEnd().
21 *
22 * This iterator should be relatively fast for sparse (few true values) arrays
23 * because it can move forward across long streches of 0s (false values) one byte
24 * (eight values) at a time rather than one value at a time.
25 *
26 * Innards: There is an issue of where to place bits in relative to bytes. Below we show an
27 * example of how our implementation uses memory for a bit array of size (in bits) 10.
28 * The digits 0-9 show the positions of bits 0-9
29 * (as accessed with operator[] for example) and "x" marks unused bit positions.
30 *
31 * In this case, a is of size two, the 16 bits starting at *a are: 7654321098xxxxxx
32 *
33 * In otherwords indices count down within a byte and count up between bytes.
34 *
35 * Initial Value: Constructor and setSize do not initialize contents.
36 */
