137=head1 Definitions
138
139By our definition of the fastafmt file format, a fastafmt stream consists of a series
140of I<records>, possibly interspersed with I<comments>. A record consists
141of a I<head>, which is one line starting with the character '>'. The head
142(sans the initial '>') and any following white space characters is considered
143to be a list of fields separated by the C<delimiterPattern>. The first
144field is called the I<name> and is sometimes considered mandatory.
145
146=head1 Constructor
147
148 FastafmtReader( std::string delimiterPattern = "[ \t]+" );
149
150The only argument is the optional C<delimiterPattern> argument, which defaults to
151C<"[ \t]+">, splitting on any continguous block of white space. To allow
152spaces in (tab separated) fields use C<FastafmtReader( "\t+" )>.
153
154=head1 Relation to other classes
155
156Some methods return records at L<LabeledAsciiSeq> and others
157L<LabeledSequence>. L<LabeledAsciiSeq> and L<LabeledSequence>. differ
158in that L<LabeledSequence> converts the sequence into a
159L<ResidueIndexMap>ed form. L<LabeledAsciiSeq> was created for the LAST
160project, while L<LabeledSequence> was created for WoLF PSORT.
161
162=head1 Reading Records

164=head2 One Record at a time

166 bool readRecord( std::istream& is ); // returns false if no more records left in stream.

168The terminal '*' character sometimes found appended to the sequences in fastafmt files,
169is removed if present.

182converts the sequence into a L<ResidueIndexMap>ed form. L<LabeledAsciiSeq> was
183created for the LAST project, while L<LabeledSequence> was created for
184WoLF PSORT.
